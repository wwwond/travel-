2번째 프로젝트 여행지 추천 사이트

사용자의 여행 기간, 선호 지역, 테마에 맞춰 최적의 여행 경로를 생성하기 위해 Scikit-learn의 K-Means 클러스터링 알고리즘을 활용한 맞춤형 추천 엔진을 구현했습니다.

1. 알고리즘 구현 : K-Means Clustering 기반 동선 최적화
지리적 군집화: 사용자가 선택한 지역 내 수많은 장소(Place)들의 위도/경도 좌표를 추출하여 K-Means 클러스터링을 수행합니다. 이를 통해 지리적으로 인접한 장소들을 하나의 그룹으로 묶어 이동 시간을 최소화하는 효율적인 동선을 보장했습니다.
평점 기반 우선순위 산출: 각 군집(Cluster) 내 장소들의 평균 평점을 계산하고, 점수가 높은 군집부터 일자별로 우선 배정하였습니다.

2. 하이브리드 추천 로직 (숙박 여부 선택)
Generate Without Accommodation: 숙박지가 정해지지 않은 경우, 여행 장소(서울 동작구 등)를 클러스터링하여 지역 내 관광지 위주로 동선을 구성하였습니다.
Generate With Accommodation: 특정 숙박지가 있는 경우, 숙소 위치를 기준점으로 설정(Radius 필터링)하여 숙소 인근의 최적 동선을 동적으로 생성합니다.

3. 기술적 특징 (Efficiency & Accuracy)
Dynamic K-Value: 여행 기간과 하루 방문 장소 수를 곱하여 클러스터 개수를 동적으로 설정하게 하였습니다.
Euclidean Distance: 군집 내에서 중심점(Centroid)과 가장 가까운 대표 장소를 선별하기 위해 유클리드 거리를 계산하여 동선의 밀집도를 높였습니다.
